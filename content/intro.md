## Definition of Terms

{% include question.html header="I. Programming and Data Basics" text="
- **Variable** ‚Äì A named container used to store data values in a program.
- **Data Type** ‚Äì The classification of data that tells Python what kind of value is being stored (e.g., integer, float, string, boolean).
- **List** ‚Äì An ordered, mutable collection of items (e.g., ```[10, 20, 30]```).
- **Dictionary** ‚Äì A collection of key-value pairs (e.g., ```{'name': 'John', 'age': 30}```).
- **Function** ‚Äì A reusable block of code that performs a specific task.
- **Module / Library** ‚Äì A collection of pre-written Python functions and tools (e.g., ```numpy```, ```pandas```, ```matplotlib```).
- **DataFrame** ‚Äì A two-dimensional, labeled data structure in ```pandas``` similar to an Excel spreadsheet, with rows and columns.
- **Series** ‚Äì A one-dimensional labeled array (e.g., a single column of a DataFrame).
- **Control Structures** ‚Äì Logical structures that determine the order in which code executes, such as loops (```for```, ```while```) and conditionals (```if```, ```else```).
" %}

{% include question.html header="II. Data Querying and Selection" text="
- **Querying** ‚Äì Extracting specific subsets of data based on certain criteria or conditions.
- **Filtering** ‚Äì Selecting rows or columns in a dataset that meet a particular condition (e.g., salaries greater than ‚Ç±80,000).
- **Indexing** ‚Äì Accessing data using row or column labels or positions.
- **iloc** ‚Äì Index-based selection (integer position).
- **loc** ‚Äì Label-based selection (using column or row names).
- **Boolean Masking** ‚Äì Using logical conditions (```True```/```False```) to filter data.
- **```isin()``` Function** ‚Äì Filters rows where a column‚Äôs value matches any in a given list.
- **String Operations** ‚Äì Text-based filters or transformations (e.g., ```str.contains()``` to find names with a certain substring).
- **```query()``` Method** ‚Äì A more readable way to filter data using string expressions (e.g., ```df.query('age > 30 and salary > 60000')```).
" %}

{% include question.html header="III. Data Transformation and Aggregation" text="
- **Grouping (```groupby```)** ‚Äì Combining rows with the same value in one or more columns to perform summary calculations.
- **Aggregation** ‚Äì Calculating summary statistics such as mean, median, count, or standard deviation for each group.
- **Custom Aggregation** ‚Äì User-defined summary functions applied to groups (e.g., calculating salary range).
- **Transformation** ‚Äì Modifying existing columns or creating new ones based on calculations or conditions.
- **Binning / Categorization** ‚Äì Dividing continuous data into categories or intervals using ```pd.cut()```.
- **Sorting** ‚Äì Arranging rows in ascending or descending order based on column values.
- **Ranking** ‚Äì Assigning a rank (1st, 2nd, etc.) based on a specific column‚Äôs value.
- **Chained Operations** ‚Äì Combining multiple DataFrame methods in a single line (e.g., filtering ‚Üí sorting ‚Üí selecting).
" %}

{% include question.html header="IV. Data Visualization" text="
- **Matplotlib** ‚Äì A powerful Python library for creating static, 2D plots like histograms, scatter plots, and bar charts.
- **Seaborn** ‚Äì A visualization library built on Matplotlib that simplifies statistical plotting with cleaner aesthetics.
- **Figure** ‚Äì The entire plotting area that may contain one or more subplots.
- **Axes / Subplot** ‚Äì Individual plots within a figure.
- **Histogram** ‚Äì Displays the frequency distribution of a numeric variable.
- **Box Plot** ‚Äì Visualizes data spread and detects outliers using quartiles.
- **Scatter Plot** ‚Äì Shows relationships or correlations between two numeric variables.
- **Bar Plot** ‚Äì Represents categorical data using rectangular bars.
- **Count Plot** ‚Äì Displays the frequency of categories.
- **Violin Plot** ‚Äì Combines box plot and density plot to show data distribution.
- **Heatmap** ‚Äì Displays a color-coded correlation matrix or table of values.
- **Pie Chart** ‚Äì Represents proportions of categories in a circular chart.
- **Legend** ‚Äì A key that identifies what each color or symbol represents in a plot.
- **Regression Line** ‚Äì A line that shows the general trend in scatter plot data (e.g., using ```sns.regplot```).
" %}

{% include question.html header="V. Descriptive and Inferential Statistics" text="
- **Statistics** ‚Äì The branch of mathematics dealing with the collection, analysis, interpretation, and presentation of data.
- **Descriptive Statistics** ‚Äì Summarize data features without drawing conclusions beyond the data itself.
- **Inferential Statistics** ‚Äì Use sample data to make generalizations about a larger population.

**üîπ Descriptive Measures**

- **Mean** ‚Äì The arithmetic average.
- **Median** ‚Äì The middle value when data are ordered.
- **Mode** ‚Äì The most frequently occurring value.
- **Range** ‚Äì Difference between the maximum and minimum values.
- **Standard Deviation (SD)** ‚Äì Measures how spread out the data are from the mean.
- **Variance** ‚Äì Square of the standard deviation.
- **Interquartile Range (IQR)** ‚Äì Difference between the 75th and 25th percentiles.
- **Skewness** ‚Äì Measures asymmetry in a distribution.
- **Kurtosis** ‚Äì Measures how heavy or light the tails of a distribution are compared to a normal distribution.
" %}

{% include question.html header="VI. Hypothesis Testing" text="
- **Null Hypothesis (H‚ÇÄ)** ‚Äì The default assumption that there is no effect or difference.
- **Alternative Hypothesis (H‚ÇÅ)** ‚Äì States that there is an effect or difference.
- **p-value** ‚Äì Probability of observing the data (or more extreme) assuming H‚ÇÄ is true.
- **Significance Level (Œ±)** ‚Äì Threshold (commonly 0.05) used to decide whether to reject H‚ÇÄ.
- **Type I Error** ‚Äì Incorrectly rejecting a true null hypothesis (false positive).
- **Type II Error** ‚Äì Failing to reject a false null hypothesis (false negative).
" %}

{% include question.html header="VII. Common Statistical Tests" text="
- **T-test** ‚Äì Compares means between groups.
  - *One-sample t-test:* compares sample mean to a known population mean.
  - *Independent (two-sample) t-test:* compares means of two independent groups.
  - *Paired t-test:* compares means of the same group before and after a treatment.
- **ANOVA (Analysis of Variance)** ‚Äì Tests differences among means of three or more groups.
  - *One-way ANOVA:** compares one factor across multiple groups.
  - *Two-way ANOVA:* examines the effect of two factors simultaneously.
  - *Post-hoc tests:* pairwise comparisons following a significant ANOVA result.
- **Chi-Square Test** ‚Äì Tests relationships between categorical variables.
  - *Test of Independence:* determines if two categorical variables are related.
  - *Goodness of Fit:* checks if observed data match expected distributions.
" %}

{% include question.html header="VIII. Correlation and Regression" text="
- **Correlation** ‚Äì Measures the strength and direction of a linear relationship between two variables (range: -1 to +1).
  - *Positive correlation:* variables increase together.
  - *Negative correlation:* one increases while the other decreases.
- **Pearson‚Äôs r** ‚Äì The most common correlation coefficient for continuous variables.
- **Regression Analysis** ‚Äì Predicts the value of a dependent variable based on one or more independent variables.
- **Simple Linear Regression** ‚Äì Uses one predictor variable to predict an outcome.
  - *Slope:* change in outcome for every one-unit increase in predictor.
  - *Intercept:* predicted outcome when the predictor is zero.
  - *R-squared:* proportion of variance in the dependent variable explained by the independent variable.
- **Logistic Regression** ‚Äì Predicts binary outcomes (e.g., ```yes/no```, ```0/1```) using continuous or categorical predictors.
  - *Odds Ratio:* how much the odds of the outcome change with a one-unit increase in the predictor.
  - *Confusion Matrix:* table showing correct and incorrect predictions.
  - *Accuracy:* proportion of correct predictions.
  - *Classification Report:* summary including precision, recall, and F1-score.
" %}

{% include question.html header="IX. Data Science Utilities" text="
- **Train-Test Split** ‚Äì Dividing data into training (for model building) and testing (for evaluation).
- **Random Seed** ‚Äì Ensures reproducibility by fixing random number generation.
- **Feature** ‚Äì An independent variable or predictor in a dataset.
- **Target Variable** ‚Äì The dependent variable or outcome being predicted.
- **Normalization / Scaling** ‚Äì Adjusting data values to a common scale without distorting differences.
" %}
